# Chrome Extension Controller (CSP-safe)

Этот вариант нужен, когда сайт блокирует `WebSocket` из консоли политикой CSP.
Расширение шлёт события так:

- `content.js`: слушает клики/мышь/клавиатуру на странице
- `background.js`: держит `WebSocket` к sync-серверу и отправляет события

## Установка (Dolphin / Chrome)

1. Откройте `chrome://extensions`
2. Включите **Developer mode**
3. Нажмите **Load unpacked**
4. Выберите папку `extension`

## Настройка адреса сервера

По умолчанию в `background.js` стоит:

- `wss://soft-production-e391.up.railway.app/ws`

Если домен другой, поменяйте `DEFAULT_SERVER_WS` в `background.js`, затем нажмите **Reload** у расширения в `chrome://extensions`.

## Проверка

1. Откройте нужный сайт на главной вкладке
2. Убедитесь, что агенты запущены (`npm run agent` на удалённых ПК)
3. Откройте панель сервера `https://<ваш-домен>/`
4. Двигайте мышь/кликайте в главной вкладке — действия должны повторяться на агентах.

## Примечание

Старый способ через вставку JS в DevTools на некоторых сайтах не работает из-за CSP.
Расширение обходит это ограничение.
